### 1. `generate_question_feedback` 함수 결과 비교

**가상 시나리오:**
*   **면접 질문:** "Tell me about a time you had a conflict with a coworker and how you resolved it." (동료와의 갈등 경험과 해결 방법에 대해 말해주세요.)
*   **사용자 답변 (Transcript):** "Yeah, so, one time my colleague, uh, he changed some code that I was working on, and it broke my feature. I was pretty annoyed. I just told him he shouldn't have done that and I fixed it myself. We didn't really talk about it after that." (네, 한번은 동료가 제가 작업하던 코드를 바꿔서 제 기능이 망가졌어요. 좀 짜증 났죠. 그냥 그에게 그러면 안 됐다고 말하고 제가 직접 고쳤습니다. 그 후로 그 일에 대해 별로 얘기 안 했어요.)

#### **(1) 기존 프롬프트 (영어 프롬프트, 영어 결과) 예시**

```json
{
  "model_answer": "When discussing conflicts, it's good to use the STAR method to structure your answer, focusing on a positive resolution and what you learned.",
  "feedback": "Your answer was direct, but it could benefit from showing more collaboration and a positive outcome."
}
```

#### **(2) 개선된 프롬프트 (영어 프롬프트, 한글 결과) 예시**

```json
{
  "model_answer": "좋은 답변은 먼저 침착하게 상황과 그로 인한 구체적인 영향을 설명하는 것에서 시작합니다. 다음으로, 개인적인 대화를 시작하는 등 동료의 관점을 이해하기 위해 취한 조치를 설명합니다. 함께 문제를 해결하기 위해 취한 협력적인 조치, 예를 들어 공유된 해결책을 찾기 위해 코드 변경 사항을 검토하는 것에 집중하세요. 더 나은 소통 프로세스를 구축한 것과 같은 긍정적인 결과를 강조하고, 그 경험을 통해 개인적으로 배운 점을 언급하며 마무리합니다.",
  "feedback": "상황을 명확하게 설명한 것은 좋은 시작이지만, 개인적인 해결보다는 협력적인 문제 해결에 더 집중한다면 답변이 훨씬 더 강력해질 것입니다."
}
```

#### **비교 분석**

| 항목 | 기존 프롬프트 (영어 결과) | 개선된 프롬프트 (한글 결과) |
| :--- | :--- | :--- |
| **프롬프트 언어** | 영어 | 영어 |
| **결과 언어** | 영어 | 한글 |
| **모범 답변** | 추상적이고 일반적인 조언 ("STAR 기법을 사용하세요") | 구체적이고 실행 가능한 단계 제시 (상황 설명 -> 동료 입장 이해 -> 협력적 해결 -> 긍정적 결과) |
| **피드백** | 다소 모호함 ("더 많은 협업을 보여주면 좋습니다") | 명확한 강점("상황을 명확히 설명한 점")과 구체적인 개선점("개인적인 해결보다 협력적인 문제 해결에 집중")을 함께 제시 |
| **유용성** | 방향성은 제시하지만, 어떻게 해야 할지 막막할 수 있음 | 사용자가 다음 답변에서 무엇을 어떻게 개선해야 할지 명확하게 인지할 수 있음 |

### 2. `generate_summary_lines` 함수 결과 비교

**가상 시나리오:**
*   사용자가 전반적으로 답변은 했지만, 답변 시간이 다소 길고 일부 답변에서 핵심을 놓치는 경향이 있었다고 가정합니다.

#### **(1) 기존 프롬프트 (영어 프롬프트, 영어 결과) 예시**

```json
[
  "You seem to have relevant experience for the role.",
  "Your average answer time was a bit long on some questions.",
  "Try to be more concise in your responses."
]
```

#### **(2) 개선된 프롬프트 (영어 프롬프트, 한글 결과) 예시**

```json
[
  "전반적인 강점: 일관적으로 침착한 태도를 유지하며 본인의 경험을 바탕으로 상세한 답변을 제공했습니다.",
  "주요 개선 영역: 답변이 직무 기술서에 언급된 특정 요구사항과 더 직접적으로 연결된다면 더 큰 영향력을 가질 수 있습니다.",
  "다음 면접을 위한 실행 가능한 팁: 답변하기 전에 잠시 시간을 갖고 STAR 기법을 사용하여 생각을 정리하면 답변의 완성도와 간결함을 모두 높일 수 있습니다."
]
```

#### **비교 분석**

| 항목 | 기존 프롬프트 (영어 결과) | 개선된 프롬프트 (한글 결과) |
| :--- | :--- | :--- |
| **프롬프트 언어** | 영어 | 영어 |
| **결과 언어** | 영어 | 한글 |
| **구조** | 구조 없이 3가지 사실을 나열 | "강점 - 개선점 - 팁"이라는 명확한 구조를 가짐 |
| **내용** | 일반적이고 피상적인 피드백 ("답변이 길다", "간결해져라") | 구체적이고 실행 가능한 피드백 ("직무 기술서와 연결", "STAR 기법 활용") |
| **유용성** | 무엇을 잘했고, 무엇을 못했고, 어떻게 해야 할지 명확하지 않음 | 사용자가 자신의 강점을 인지하고, 약점을 보완할 구체적인 방법을 얻을 수 있음 |
